## Task
Improve `src/main.rs` based on vibecheck findings.

## Findings (top 20)
- [high] consistency — `src/main.rs:99-278`
  - The `--run` agent spawn logic is copy-pasted verbatim between the `Create` and `Create { subcommand: Next }` arms — ~25 identical lines each loading config, checking `config.run`, building the command, spawning via `sh -c`, and handling exit codes. Any bug fix must be applied in both places.
  - Suggested fix: Extract a shared `spawn_agent_for_bean(beans_dir, bean_id)` helper function and call it from both code paths.
- [medium] function-length — `src/main.rs:56-695`
  - `main()` is a single 640-line function consisting of a giant `match cli.command` with 30+ arms, many containing significant inline logic (stdin resolution, validation, config loading, agent spawning). Adding a new command requires modifying this already-long match.
  - Suggested fix: Each match arm should call a thin adapter function that converts CLI args to domain args. Move the `Create` arm's ~200 lines of logic (interactive detection, stdin resolution, json output, --run spawning) into `commands/create.rs` as an entry point.

## Notes
- Keep behavior the same unless the finding explicitly calls for a behavior change.
- Prefer small, focused changes.
