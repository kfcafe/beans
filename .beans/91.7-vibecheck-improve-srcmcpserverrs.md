## Task
Improve `src/mcp/server.rs` based on vibecheck findings.

## Findings (top 20)
- [medium] input-validation — `src/mcp/server.rs:73-78`
  - The MCP `dispatch` function passes raw `params` from JSON-RPC requests directly to tool handlers without any schema validation. While individual tool handlers in `tools.rs` may validate their specific fields, there's no central validation against the `input_schema` defined in `tool_definitions()`. Malformed or extra parameters are silently accepted.
  - Suggested fix: Add schema validation in the `handle_tools_call` dispatcher before calling individual tool handlers. At minimum, validate required fields are present and types match. The `input_schema` is already defined — use it.
- [low] input-validation — `src/mcp/server.rs:39-53`
  - The MCP server reads newline-delimited JSON from stdin with no maximum line length. A malicious client could send an extremely long single line, causing unbounded memory allocation during the `reader.lines()` call before JSON parsing even begins.
  - Suggested fix: Use `BufRead::read_line()` with a size limit instead of the `lines()` iterator, or wrap stdin in a `BufReader` with a bounded buffer and reject lines exceeding a reasonable limit (e.g., 10 MB).

## Notes
- Keep behavior the same unless the finding explicitly calls for a behavior change.
- Prefer small, focused changes.
