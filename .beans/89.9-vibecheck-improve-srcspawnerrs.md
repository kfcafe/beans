## Task
Improve `src/spawner.rs` based on vibecheck findings.

## Findings (top 20)
- [medium] input-validation — `src/spawner.rs:127-137`
  - The spawner constructs shell commands by string substitution (`template.replace("{id}", bean_id)`) and passes them to `sh -c`. While bean IDs are validated against `[a-zA-Z0-9._-]+`, the config templates (`config.run`, `config.plan`) are user-controlled strings from `config.yaml`. A malicious config template (e.g., from `extends` or a cloned repo) could contain arbitrary shell commands that execute with user privileges when `bn run` is invoked.
  - Suggested fix: Document the security implications of `extends` configs from untrusted sources. Consider validating that templates contain `{id}` and warning if they contain suspicious patterns (`;`, `&&`, backticks, `$()` outside the template placeholder). At minimum, add a trust check before executing templates from extended configs.
