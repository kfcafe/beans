## Task
Improve `src/spawner.rs` based on vibecheck findings.

## Findings (top 20)
- [medium] dead-code — `src/spawner.rs:1-683`
  - The `Spawner` struct and its methods (`spawn`, `check_completed`, `kill_all`, `list_running`, etc.) are never used outside of `spawner.rs` itself (only in its own tests). The `log_dir`, `find_latest_log`, and `find_all_logs` functions are simple delegation wrappers to `commands::logs`. The `AgentAction` enum is a duplicate of `commands::run::BeanAction`. The entire module appears to be legacy code superseded by the direct-mode spawning in `commands/run.rs`.
  - Suggested fix: Verify that `Spawner` is truly unused outside tests, then remove it or mark it as deprecated with a note pointing to the direct-mode spawning in `commands/run.rs`. Remove the delegation wrappers (`log_dir`, `find_latest_log`, `find_all_logs`) and have callers use `commands::logs` directly. Consolidate `AgentAction` and `BeanAction` into a single enum.
- [medium] input-sanitization — `src/spawner.rs:81-82`
  - `substitute_template` performs simple string replacement of `{id}` in command templates. If a bean ID contains shell metacharacters (currently prevented by `validate_bean_id`), this would be a command injection vector. The defense-in-depth is weak — validation happens at the CLI/MCP boundary but not at the point of use. If a new entry point bypasses `validate_bean_id`, the shell injection path opens up.
  - Suggested fix: Add a redundant `validate_bean_id` check inside `substitute_template` or use `shell-escape` (already a dependency) to quote the bean ID before substitution: `template.replace("{id}", &shell_escape::escape(bean_id.into()))`.

## Notes
- Keep behavior the same unless the finding explicitly calls for a behavior change.
- Prefer small, focused changes.
