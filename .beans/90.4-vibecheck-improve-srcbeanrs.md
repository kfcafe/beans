## Task
Improve `src/bean.rs` based on vibecheck findings.

## Findings (top 20)
- [medium] function-length — `src/bean.rs:315-348`
  - `Bean::try_new` initializes ~30 fields with defaults. While not complex, any new field added to `Bean` requires updating this constructor, `from_string`, and `apply_value` — three places that must stay in sync manually with no compiler enforcement.
  - Suggested fix: Consider using `#[derive(Default)]` for optional fields and building on top of a default value. Alternatively, use the builder pattern or `..Default::default()` syntax to reduce the maintenance burden when adding new fields.
- [medium] architecture — `src/bean.rs:525-561`
  - `apply_value` is a 35-arm match on field names as strings. Adding a new field requires updating this match, `try_new`, and the serde derives — three separate locations with no compile-time verification that they're in sync. If a field is added to the struct but missed in `apply_value`, it silently fails with "Unknown field".
  - Suggested fix: Consider generating `apply_value` with a macro that iterates over struct fields, or at minimum add a test that verifies every serializable field is handled by `apply_value` (e.g., by serializing a bean to JSON and checking each key is accepted).
- [low] comprehensibility — `src/bean.rs:162-197`
  - The `Bean` struct has 30+ fields with varying `#[serde(...)]` attributes. While each field is documented, the sheer density makes it hard to see the logical groupings. Fields are loosely ordered but not separated by concern (e.g., verification fields, memory fields, and metadata fields are interspersed).
  - Suggested fix: Add section comments (like the existing `// -- verification & claim fields --` and `// -- Memory system fields --`) more consistently. Group related fields together: core identity (id, title, status), content (description, acceptance, notes, design), relationships (parent, dependencies, produces, requires), verification (verify, fail_first, attempts, history), and metadata (tokens, bean_type, paths, attempt_log).
