## Task
Improve `src/main.rs` based on vibecheck findings.

## Findings (top 20)
- [medium] function-length — `src/main.rs:49-695`
  - `main()` is ~646 lines — a single giant match statement with inline argument marshalling, stdin resolution, validation, and error message construction duplicated across branches (e.g., the `--run` spawn logic appears in both `Create` and `Create::Next` arms).
  - Suggested fix: The Create + Create::Next branches share ~40 lines of identical `--run` spawning logic. Extract a `spawn_run_for_bean(beans_dir, bean_id, config)` helper. Consider grouping related match arms into dispatch functions (e.g., `handle_create_command(...)`, `handle_dep_command(...)`) to flatten the nesting.
- [medium] consistency — `src/main.rs:100-224`
  - The `Create::Next` subcommand handling in main.rs (lines ~100-224) duplicates nearly the entire `Create` flow — stdin resolution, title validation, on_fail parsing, cmd_create call, JSON output, and `--run` spawning — instead of reusing the same code path. Meanwhile `cmd_create_next` exists in create.rs but is never called from main.rs.
  - Suggested fix: Use `cmd_create_next` from main.rs for the `Create::Next` arm, or refactor the shared create logic into a single helper that both arms call. The current duplication means bugs fixed in one path may not be fixed in the other.

## Notes
- Keep behavior the same unless the finding explicitly calls for a behavior change.
- Prefer small, focused changes.
