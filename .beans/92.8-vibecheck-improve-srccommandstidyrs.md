## Task
Improve `src/commands/tidy.rs` based on vibecheck findings.

## Findings (top 20)
- [medium] edge-cases — `src/commands/tidy.rs:77-94`
  - `has_running_agents()` uses `pgrep -f` with patterns derived from the run command template. This is fragile: it matches any process containing the pattern string (false positives from editor windows, documentation processes, etc.). It also hardcodes fallback patterns `"pi -p beans"`, `"deli spawn"`, `"claude"` which are stale references to deprecated tooling.
  - Suggested fix: Use the `.beans/agents.json` file (which tracks running agents with PIDs) instead of `pgrep`. The agents file is already maintained by `bn run` and `bn agents`. The `pgrep` fallback patterns should be removed or made configurable.

- Keep behavior the same unless the finding explicitly calls for a behavior change.
- Prefer small, focused changes.
