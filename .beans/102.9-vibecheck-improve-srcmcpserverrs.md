## Task
Improve `src/mcp/server.rs` based on vibecheck findings.

## Findings (top 20)
- [medium] test-coverage — `src/mcp/server.rs:1-234`
  - Zero tests for the MCP JSON-RPC server loop. No validation that request parsing, dispatch, notification handling, or error responses work correctly. Protocol compliance is untested.
  - Suggested fix: Add tests that send JSON-RPC messages through the server and verify responses, including error cases (malformed JSON, unknown methods, missing params).
- [low] input-validation — `src/mcp/server.rs:30-35`
  - The MCP server reads line-delimited JSON-RPC from stdin with `reader.lines()` which has no message size limit. A malicious MCP client could send an extremely large JSON line to exhaust memory. While MCP clients are typically trusted IDE extensions, this is still a resource exhaustion vector.
  - Suggested fix: Replace `reader.lines()` with a size-limited read: use `BufRead::read_line()` with a check on buffer size (e.g., 10MB limit), or wrap stdin in a `Take` reader to cap individual message sizes.

## Notes
- Keep behavior the same unless the finding explicitly calls for a behavior change.
- Prefer small, focused changes.
