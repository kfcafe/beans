## Task
Improve `src/config.rs` based on vibecheck findings.

## Findings (top 20)
- [medium] edge-cases — `src/config.rs:131-162`
  - Config inheritance (`load_with_extends`) detects default values by comparing against hardcoded default functions (e.g., `config.max_tokens == default_max_tokens()`). This means if a user explicitly sets `max_tokens: 30000` (the default value) in their local config, it will be overridden by a parent config's different value. The user's explicit choice is silently ignored because it happens to match the default.
  - Suggested fix: Track which fields were explicitly set (e.g., using `Option<T>` for inheritable fields, or a separate "dirty" bitflag) so that explicit-but-default values aren't overridden. Alternatively, document this behavior clearly as a known limitation.
- [medium] input-validation — `src/config.rs:133-148`
  - `resolve_extends_path` allows `~/` expansion and relative paths for config inheritance. A malicious `.beans/config.yaml` with `extends: ["../../sensitive/config.yaml"]` could read arbitrary YAML files outside the project as config, potentially leaking information (the parsed fields are limited, but error messages may disclose file contents). There's no validation that the resolved path stays within the project directory.
  - Suggested fix: After resolving the path, canonicalize it and verify it's either within the project root or within `~/.config/beans/` (or similar expected locations). At minimum, warn when extends paths resolve outside the project.
- [low] input-validation — `src/config.rs:48-49`
  - The `rules_file` config field accepts an absolute path (`if p.is_absolute() { p.to_path_buf() }`) which allows reading arbitrary files on the filesystem via `bn context`. The file content is embedded into agent prompts.
  - Suggested fix: Reject absolute paths in `rules_file` or validate they're within the project directory. This is a local-only tool so the risk is low, but the pattern could be dangerous if configs are shared or synced.

## Notes
- Keep behavior the same unless the finding explicitly calls for a behavior change.
- Prefer small, focused changes.
