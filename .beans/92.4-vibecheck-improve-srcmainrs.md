## Task
Improve `src/main.rs` based on vibecheck findings.

## Findings (top 20)
- [medium] function-length — `src/main.rs:52-695`
  - The `main()` function is ~640 lines — a giant match statement that destructures CLI args, resolves selectors, validates IDs, and dispatches to commands. The `Command::Create` branch alone is ~200 lines with duplicated `--run` handling between the `Create` and `Create::Next` subcommands.
  - Suggested fix: Move per-command argument resolution into each command module (e.g., `cmd_create` should accept raw CLI args and handle stdin resolution, @latest, --run internally). Main should be a thin dispatcher: `Command::Create { args } => cmd_create(&beans_dir, args)`.
- [medium] consistency — `src/main.rs:99-301`
  - The `--run` agent spawning logic (load config, get template, substitute, spawn shell) is copy-pasted between the `Command::Create` branch (lines ~210-237) and the `Command::Create::Next` branch (lines ~99-130). Both blocks are identical except for variable scoping.
  - Suggested fix: Extract into a shared helper function like `spawn_agent_for_bean(beans_dir, bean_id)` and call it from both branches.

## Notes
- Keep behavior the same unless the finding explicitly calls for a behavior change.
- Prefer small, focused changes.
