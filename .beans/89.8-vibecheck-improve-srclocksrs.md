## Task
Improve `src/locks.rs` based on vibecheck findings.

## Findings (top 20)
- [medium] resource-cleanup — `src/locks.rs:1-80`
  - File locks are created as JSON files on disk but have no `Drop` implementation or RAII guard. If an agent panics or is killed (SIGKILL), locks remain as stale files forever. The stale-PID check exists for `acquire`, but there's no periodic cleanup or lock expiry mechanism — locks from dead processes accumulate until someone manually runs cleanup or the next `acquire` check detects the stale PID.
  - Suggested fix: Add a `LockGuard` wrapper that releases the lock on `Drop`. Additionally, add a TTL or last-heartbeat timestamp to lock files so `bn doctor` or startup can garbage-collect truly stale locks.
