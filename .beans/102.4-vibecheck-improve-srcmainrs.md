## Task
Improve `src/main.rs` based on vibecheck findings.

## Findings (top 20)
- [medium] function-length — `src/main.rs:47-709`
  - `main()` is 660+ lines, consisting almost entirely of a single giant `match` on command variants. The `Command::Create` arm alone is ~200 lines with inline business logic (stdin resolution, interactive mode branching, `--run` spawning). The `--run` spawning block is copy-pasted between `Command::Create` and the `CreateSubcommand::Next` path.
  - Suggested fix: Move command-specific logic into the command modules. Each match arm should be 1-5 lines that parse args and delegate. The `--run` spawning logic should be extracted into a shared helper (it's duplicated at lines 152-175 and 220-240).
- [medium] function-length — `src/main.rs:50-530`
  - `main()` is a single ~480-line function consisting of one massive match statement. The `Command::Create` arm alone is ~200 lines. The `--run` agent spawning logic (load config, check template, substitute `{id}`, spawn via `sh -c`) is duplicated between the `Create` handler (lines ~200-215) and the `Create::Next` handler (lines ~160-175) — nearly identical blocks of ~15 lines each.
  - Suggested fix: Extract the duplicated `--run` spawning logic into a helper: `fn spawn_agent_for_bean(beans_dir: &Path, bean_id: &str) -> Result<()>`. Call it from both Create and Create::Next. Consider extracting each major match arm into its own handler function to reduce the main function size.
- [medium] consistency — `src/main.rs:155-215`
  - The `--run` agent spawning code is copy-pasted between the `Create::Next` subcommand handler and the main `Create` handler. Both blocks do: load config → check `config.run` → substitute `{id}` → spawn via `sh -c` → handle exit status. Any change to spawning behavior must be made in both places. The same pattern also exists in `src/spawner.rs` (which shells out to `bn` itself).
  - Suggested fix: Consolidate into a single `spawn_from_config(beans_dir, bean_id)` function and call it from all three locations.

## Notes
- Keep behavior the same unless the finding explicitly calls for a behavior change.
- Prefer small, focused changes.
