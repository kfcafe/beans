## Task
Improve `src/commands/agents.rs` based on vibecheck findings.

## Findings (top 20)
- [medium] edge-cases — `src/commands/agents.rs:69-70`
  - `process_alive` uses `unsafe { libc::kill(pid as i32, 0) == 0 }` but doesn't account for `EPERM` (permission denied). A process may be alive but owned by another user — `kill(0)` returns -1 with `errno=EPERM` in that case. This function would incorrectly report the process as dead. The same logic in `locks.rs:250` correctly handles `EPERM`.
  - Suggested fix: Use the same pattern as `locks.rs:is_process_alive` which checks `if ret == 0 { true } else { last_os_error == EPERM }`. Ideally, extract to a shared `util::is_process_alive` function.
