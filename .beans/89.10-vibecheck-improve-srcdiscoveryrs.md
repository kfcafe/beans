## Task
Improve `src/discovery.rs` based on vibecheck findings.

## Findings (top 20)
- [low] performance — `src/discovery.rs:50-62`
  - `find_bean_file` uses `glob::glob` to scan the entire `.beans/` directory for every bean lookup, including during batch operations that look up many beans. The glob pattern `{beans_dir}/*{id}-*.md` could also false-match if an ID is a substring of another ID (e.g., looking for "1" could match "11-slug.md" via the `*{id}-*` pattern, though the subsequent `starts_with` check prevents this).
  - Suggested fix: Since bean file naming is deterministic (`{id}-*.md`), use `fs::read_dir` with a prefix filter instead of glob. For batch operations, build a filename→path map once and reuse it.

## Notes
- Keep behavior the same unless the finding explicitly calls for a behavior change.
- Prefer small, focused changes.
