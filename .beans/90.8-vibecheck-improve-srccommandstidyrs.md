## Task
Improve `src/commands/tidy.rs` based on vibecheck findings.

## Findings (top 20)
- [medium] edge-cases — `src/commands/tidy.rs:65-93`
  - `has_running_agents()` uses `pgrep -f` to detect running agents by searching process names. This is fragile: it can match unrelated processes (e.g., an editor with "claude" in a filename), and on systems without `pgrep` it silently returns false. The fallback patterns ("pi -p beans", "deli spawn", "claude") are hardcoded and may not match actual agent processes.
  - Suggested fix: Use PID tracking — when `bn run` spawns agents, record their PIDs in `.beans/agents.json` (which already exists). Check if those PIDs are alive instead of pattern-matching process names. This is more reliable and doesn't require guessing process patterns.
