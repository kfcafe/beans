## Problem

`bean.to_file()` and `index.save()` write directly to the target path. A crash or kill mid-write corrupts the file.

## Approach

Write to a temp file in the same directory, then `rename()` atomically.

### Files to modify
- `src/util.rs` — add `atomic_write(path, contents)` helper
- `src/bean.rs` — `to_file()` uses atomic_write
- `src/index.rs` — `save()` uses atomic_write

### Design
- Write to `{path}.tmp.{pid}` then `std::fs::rename()`
- rename() is atomic on POSIX (same filesystem guaranteed since same dir)
- Clean up temp file on error
- No new dependencies needed
