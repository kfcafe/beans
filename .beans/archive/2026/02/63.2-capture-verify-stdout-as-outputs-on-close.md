## Task
When verify succeeds, capture stdout and store as bean.outputs (JSON if parseable).

## Steps
1. In src/commands/close.rs, modify the verify success path (around line 300):
   - Capture stdout separately from stderr (current run_verify combines them)
   - Update `run_verify` to return stdout and stderr separately:
   ```rust
   struct VerifyResult {
       success: bool,
       exit_code: Option<i32>,
       stdout: String,
       stderr: String,
       output: String,  // combined, for backward compat
   }
   ```

2. After verify passes, try to parse stdout as JSON:
   ```rust
   if verify_result.success {
       if !verify_result.stdout.trim().is_empty() {
           match serde_json::from_str::<serde_json::Value>(verify_result.stdout.trim()) {
               Ok(json) => {
                   bean.outputs = Some(json);
               }
               Err(_) => {
                   // Not JSON — store as text wrapper
                   bean.outputs = Some(serde_json::json!({
                       "text": verify_result.stdout.trim()
                   }));
               }
           }
       }
   }
   ```

3. Cap output size: if stdout > 64KB, truncate and store as text with warning

## Files
- src/commands/close.rs (update VerifyResult, add output capture on success)

## Tests
- JSON stdout stored as outputs
- Non-JSON stdout stored as { text: "..." }
- Empty stdout results in no outputs
- Large stdout (>64KB) truncated with warning
- Stderr does NOT get captured as outputs
- Existing failure behavior unchanged
