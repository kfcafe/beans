## What

Add `bn race <id>` command that dispatches N agents on the same bean in parallel, waits for all to finish, and lets the human pick the best implementation. Race is about producing the best code, not finishing first.

## Why

From HN (_pdp_): "Spin up five agents to implement five different versions and simply pick the best one." But finishing first ≠ best code. All candidates should complete, then the human reviews and picks.

## Steps

1. Add `Race` variant to `Command` enum in `src/cli.rs`:
   - `bn race <id>` — race on a specific bean
   - `--copies N` (default 3) — how many agents to spawn
   - `--models "claude,gemini"` — optionally use different models per candidate
   - `--timeout <minutes>` — max time per candidate
2. Create `src/commands/race.rs` with:
   - `cmd_race()` — creates N worktrees (one per candidate), spawns an agent in each, waits for all to complete
   - Each candidate gets its own git branch: `race/<bean_id>/candidate-<n>`
   - Track which candidates passed verify
3. Add `bn race pick <id>` subcommand:
   - Shows all candidates: diff stat, lines changed, verify result (pass/fail)
   - Human selects winner
   - Winner's branch gets merged, losers get cleaned up
4. Add `Race` to `src/commands/mod.rs` and wire into `main.rs`

## Acceptance Criteria
- `bn race <id> --copies 3` spawns 3 agents in separate worktrees
- All agents run to completion (or timeout), not first-wins
- `bn race pick <id>` shows candidates with diff stats
- Selected winner's changes merge back, losers cleaned up
- Works with existing config.yaml run templates
