## What

Add a configurable timeout for verify command execution. If verify takes longer than the limit, kill the process and treat it as a failure.

## Steps

1. Add verify_timeout field to Bean struct (optional u64, seconds). Pattern: same as stale_after or max_loops optional fields.
2. Add verify_timeout to Config as project-wide default.
3. Find where verify commands are executed in close.rs and verify.rs — look for Command::new or process spawning. Add timeout logic: spawn child, wait with timeout, kill on expiry. Bean-level overrides config-level.
4. On timeout: return failure with message 'Verify timed out after Ns'.
5. Add --verify-timeout flag to Create and Quick CLI commands.
6. Add unit tests.

## Key patterns to follow
- Optional fields use skip_serializing_if = Option::is_none
- Config defaults use default = fn_name pattern
- Verify execution likely uses std::process::Command — add wait_with_timeout logic

## Don't
- Don't add external timeout crate — use std::process with wait/kill
- Don't change verify semantics — timeout is just another failure mode
- Don't restructure close.rs — surgical addition only
