## What

Add `bn trace <id>` command that shows the full lineage and dependency chain for a bean — parent, children, dependencies, dependents, what it produces/requires, attempt history summary.

## Why

From VSDD's "contract chain" and DaylitMagic's comment about sequencing. Being able to answer "why does this bean exist and what led to it?" is valuable for understanding complex decompositions.

## Steps

1. Add `Trace` variant to `Command` enum in `src/cli.rs`:
   - `bn trace <id>` — show full trace
   - `--json` — machine-readable output
2. Create `src/commands/trace.rs`:
   - `cmd_trace()` — loads the bean and walks the graph:
     - Parent chain (up to root)
     - Children (direct)
     - Dependencies (what this bean waits on)
     - Dependents (what waits on this bean)
     - Produces/requires artifacts
     - Attempt summary (N attempts, M tokens, pass/fail)
3. Output format: indented tree showing relationships

Example output:
```
Bean 7.3: "Implement rate limiter" [closed]
  Parent: 7 "API hardening" [open]
    Parent: (root)
  Dependencies:
    → 7.1 "Define rate limit types" [closed]
    → 7.2 "Add redis client" [closed]
  Dependents:
    ← 7.4 "Integration tests" [open, blocked]
  Produces: rate_limiter.rs, RateLimiter type
  Requires: RateLimitConfig (from 7.1)
  Attempts: 2 (45K tokens, $0.18)
```
