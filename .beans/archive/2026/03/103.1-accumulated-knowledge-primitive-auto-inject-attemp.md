## What

Make attempt_log a first-class knowledge accumulator. When an agent is dispatched for a bean, previous attempt notes are automatically included in the context — the agent starts fresh but inherits lessons from predecessors.

## Why

Currently attempt_log exists but agents don't see it unless someone manually reads the bean. This means retry agents repeat the same mistakes. The HN discussion highlighted "prototype onion" — each attempt builds on the last.

## Steps

1. In `src/commands/context.rs`, modify `cmd_context()` to include a "Previous Attempts" section when the bean has attempt_log entries with notes
2. Format: show attempt number, agent name, outcome, and notes for each previous attempt
3. Add test: create a bean with attempt_log entries containing notes, verify `bn context <id>` output includes them
